# 风格写作用户指南

## 使用前准备

服务端需要完成数据库迁移，并配置以下环境变量：

- `DATABASE_URL`：PostgreSQL 15+ 连接地址
- `OPENAI_API_KEY`：用于风格向量和内容生成
- `REDIS_URL`：LLM 限流和异步任务使用

质量阈值可通过 `STYLE_AI_FLAVOR_THRESHOLD`、`STYLE_SIMILARITY_THRESHOLD` 和
`STYLE_GENERATION_MAX_RETRIES` 调整；默认值分别为 `0.7`、`0.75` 和 `2`。
`STYLE_GENERATION_TIMEOUT_SECONDS` 控制单次生成总预算，默认 30 秒。

首次部署或本次版本更新后执行：

```bash
cd QuillMind/backend
python manage.py migrate
```

## 创建风格档案

1. 注册并登录文墨。
2. 从左侧导航进入「风格写作」。
3. 点击「创建风格档案」，输入容易识别的名称。
4. 粘贴或上传至少 3 篇写作样本。支持 `txt`、`md` 和 `docx`，每篇至少
   100 个非空白字符，单个文件不超过 2 MB。
5. 点击「创建并分析」，等待系统提取风格特征。

建议使用同一作者、同一写作场景的 3–5 篇完整文本。请只上传本人作品或已获授权
的内容；聊天记录应先删除姓名、电话、地址等敏感信息。

## 查看和维护档案

档案详情页会展示：

- 高频词云
- 平均句长和句长波动
- 常用语气词
- 感叹号、省略号等标点习惯
- 可展开查看的原始样本

可以在详情页编辑名称、追加样本或删除档案。追加样本后，系统会重新计算全部特征
和风格向量。

## 按风格生成内容

1. 在档案详情页点击「开始写作」。
2. 填写主题或写作任务；可补充大纲和关键词。
3. 拖动「正式克制 ↔ 自然口语」滑块。
4. 点击「开始生成」。内容会流式显示；若去 AI 味或风格相似度不达标，系统最多
   自动优化两次。
5. 生成完成后可复制全文，并用 👍 / 👎 提交反馈。

页面底部保存该档案最近的生成记录，可以展开、复制，或载入历史主题再次生成。

## MVP 联调验收

仓库提供 `scripts/style_mvp_e2e.py`，会在指定的开发或测试环境创建 3 个一次性测试
账号，使用 3 组不同文风样本，自动执行：

`注册 → 登录 → 创建档案 → 流式生成 → 评价 → 查询历史`

运行方式：

```bash
export WENMO_BASE_URL="https://your-dev-host.example.com"
export WENMO_E2E_PASSWORD="仅用于本次测试的临时密码"
python scripts/style_mvp_e2e.py > style-mvp-report.json
```

脚本会检查：

- 每个档案由 3 篇样本创建
- 流式首 token 不超过 3 秒
- 总生成时间不超过 30 秒
- 反馈成功保存
- 生成结果出现在历史记录

输出报告包含临时测试账号邮箱、生成耗时和质量指标，不包含密码。验收结束后应从
开发数据库删除 `style-e2e-` 前缀账号。三组内置文本仅用于功能联调；正式产品验收
应换成已获授权的真实样本，并由产品负责人评估生成内容是否保留原作者的表达习惯。

## 常见问题

### 创建档案提示“风格提取服务暂时不可用”

检查 `OPENAI_API_KEY` 是否有效，以及部署环境能否访问模型服务。

### 生成后自动出现第二版或第三版

初稿未达到去 AI 味或风格相似度阈值，系统正在自动优化。这是正常的质量控制流程。

### 流式生成长时间没有文字

检查模型服务延迟、反向代理是否关闭响应缓冲，以及响应头是否保留
`text/event-stream`。Nginx 应尊重后端返回的 `X-Accel-Buffering: no`。

### 页面显示 401 或自动回到登录页

访问令牌已过期且刷新失败。重新登录后再次操作。
